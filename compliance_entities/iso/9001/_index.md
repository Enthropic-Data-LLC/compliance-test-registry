# ISO 9001:2015 — Quality Management System

**Registry version:** 2026.05
**Last updated:** 2026-05-20
**Scope:** All clauses 4–10; universal QMS applicable to any organization, any size, any sector
**Authority:** International Organization for Standardization (ISO)
**Enforcing context:** Third-party certification (accredited certification body); contractual requirements; foundation for sector-specific derivatives (ISO 13485, IATF 16949, AS9100, ISO/TS 22163, etc.)
**Current edition:** ISO 9001:2015 (uses Annex SL High Level Structure — harmonized with ISO 14001, ISO 27001, ISO 45001)
**Note:** ISO 9001 is the parent QMS standard. All sector-specific QMS standards in this registry (ISO 13485, IATF 16949, AS9100) are built on this foundation with additional requirements. Understanding 9001 is prerequisite to all of them.

## Risk-based thinking (RBT) — the central design principle

ISO 9001:2015 eliminated the requirement for a formal risk register but introduced "risk-based thinking" throughout. The organization determines what is appropriate for their context.

**RDF treatment:** RBT itself is CONTESTED (no methodology specified). However, the outputs of RBT — objectives, documented information, process controls, corrective actions — are DETERMINISTIC/PARAMETERIZED and are what auditors verify. The tests target the outputs, not the method.

## CI/CD gate configuration

Standard three-tier gate (see NERC CIP registry). ISO 9001-specific constraints:

- **Record existence checks (Pattern 1):** The 11 required record types in the table above are binary checks — record must exist for each transaction/process instance where it is required.
- **Calibration currency (Pattern 1):** Each measurement device used in product conformity testing must have a calibration record with a current calibration date.
- **Internal audit schedule (Pattern 1):** Annual audit program must be documented and completed; overdue audits trigger Pattern 2.
- **Nonconformity closure (Pattern 2):** Open nonconformities past their committed closure date trigger Pattern 2 escalation.
- **Management review recency (Pattern 1):** Management review must occur at planned intervals; > 12 months since last review triggers Pattern 2.
