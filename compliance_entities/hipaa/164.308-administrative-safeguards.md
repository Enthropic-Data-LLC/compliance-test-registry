## Scope summary

§164.308 is the largest and most complex HIPAA Security Rule section. It covers nine standards spanning workforce management, access control, training, incident response, contingency planning, and ongoing evaluation. The dominant challenge is the "reasonable and appropriate" language that pervades the addressable specifications — there is no HHS-defined floor for most of them. OCR audit results and Resolution Agreements provide the closest thing to enforcement precedent, and those are used to inform the assumption registry.

**Addressable reminder:** Addressable ≠ optional. For each addressable specification, the covered entity must either (a) implement it, or (b) document why it is not reasonable and appropriate AND implement an equivalent alternative. Test code for addressable specifications checks for one of those two conditions — not just implementation.

        restore_tests = [r for r in records if r.get("is_restore_test")]
        if not restore_tests:
            violations.append(f"{system_id}: no restore test on record")
            continue
        most_recent = max(restore_tests, key=lambda r: r["backup_date"])
        age = datetime.utcnow().date() - most_recent["backup_date"]
        if age.days > BACKUP_RESTORE_TEST_INTERVAL_DAYS:
            violations.append(
                f"{system_id}: last restore test {age.days} days ago "
                f"(threshold {BACKUP_RESTORE_TEST_INTERVAL_DAYS})"
            )
    assert not violations, (
        f"VIOLATION: {len(violations)} system(s) overdue for restore test:\n"
        + "\n".join(violations)
    )


@pytest.mark.assumption(id="ASSUME-308-007")
def test_backup_media_encrypted(backup_records):
    violations = [
        r for r in backup_records
        if not r.get("storage_encrypted")
    ]
