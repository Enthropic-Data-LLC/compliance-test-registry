## Scope summary

§164.310 requires physical safeguards for all facilities and workstations that access ePHI, as well as controls over devices and media that store or contain ePHI. It is the shortest HIPAA Security Rule section but generates significant enforcement activity because physical controls are often neglected in favor of technical ones. Workstation security is almost entirely PARAMETERIZED — HIPAA does not specify lock types, badge systems, or screen positions; it requires that appropriate physical measures be taken.

"""
164.310(a)(1) — Facility Access Controls (Addressable)
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED (ASSUME-310-001)
"""
import pytest
from datetime import datetime, timedelta

ACCESS_LIST_REVIEW_INTERVAL_DAYS = 90
ACCESS_LOG_REVIEW_INTERVAL_DAYS = 31


@pytest.mark.assumption(
    id="ASSUME-310-001",
    description=(
        "Adequate: authorized list reviewed quarterly, access mechanism in place, "
        "access events logged with identity and timestamp, logs reviewed monthly"
    ),
    approved_by="Compliance Officer",
    review_date="2026-05-20",
)
def test_facility_access_controls_implemented(facility_access_controls, ephi_facilities):
    controlled_facility_ids = {f["facility_id"] for f in facility_access_controls}
    uncontrolled = [
        f for f in ephi_facilities
        if f["facility_id"] not in controlled_facility_ids
    ]
    assert not uncontrolled, (
        f"VIOLATION: {len(uncontrolled)} ePHI facility/facilities with no "
        f"access control record: {[f['facility_id'] for f in uncontrolled]}"
    )


@pytest.mark.assumption(id="ASSUME-310-001")
def test_authorized_access_list_reviewed_quarterly(facility_access_controls):
    violations = []
    for fac in facility_access_controls:
        last_review = fac.get("access_list_last_reviewed_date")
        if not last_review:
            violations.append(f"{fac['facility_id']}: no access list review date")
            continue
        age = datetime.utcnow().date() - last_review
        if age.days > ACCESS_LIST_REVIEW_INTERVAL_DAYS:
            violations.append(
                f"{fac['facility_id']}: access list last reviewed {age.days} days ago"
            )
    assert not violations, (
        f"VIOLATION: {len(violations)} facility/facilities with stale access list:\n"
        + "\n".join(violations)
    )


@pytest.mark.assumption(id="ASSUME-310-001")

import pytest

ACCEPTABLE_SANITIZATION_METHODS = {
    "NIST_800-88_Clear",
    "NIST_800-88_Purge",
    "NIST_800-88_Destroy",
    "DoD_5220.22-M",
    "cryptographic_erase",
    "physical_destruction",
    "degauss",

@pytest.mark.assumption(
    id="ASSUME-310-005",
    description="Acceptable methods: NIST SP 800-88 Clear/Purge/Destroy; certificate of destruction for vendor disposal",
    approved_by="Compliance Officer",
    review_date="2026-05-20",
)
def test_all_disposed_ephi_assets_have_sanitization_record(
    disposed_assets, disposal_records
):
    disposal_asset_ids = {r["asset_id"] for r in disposal_records}
    missing = [
        a for a in disposed_assets
        if a.get("stored_ephi")
        and a["asset_id"] not in disposal_asset_ids
    ]
    assert not missing, (
        f"VIOLATION: {len(missing)} disposed ePHI asset(s) with no sanitization record: "
        f"{[a['asset_id'] for a in missing]}"
    )


@pytest.mark.assumption(id="ASSUME-310-005")
def test_disposal_uses_acceptable_sanitization_method(disposal_records):

        if r.get("sanitization_method") not in ACCEPTABLE_SANITIZATION_METHODS
    ]

        f"VIOLATION: {len(violations)} disposal record(s) use an unacceptable "
        f"sanitization method: "

## Notes for the registry

- **Remote work ePHI access:** §164.310(c) applies to any workstation accessing ePHI, including employee home workstations and personal devices under BYOD policies. The workstation security record must account for remote endpoints; VPN-only access, endpoint MDM enforcement, and screen lock policies are the standard controls.
- **PHI on paper:** Physical safeguards for paper PHI are governed by the Privacy Rule, not the Security Rule. §164.310 applies exclusively to electronic PHI. However, printed ePHI (e.g., printed from an EHR) becomes paper PHI at that moment — the workstation use policy should address printing controls.
- **Cloud-hosted ePHI:** If ePHI is hosted in a cloud environment, §164.310 obligations fall on the covered entity for the aspects they control (e.g., their own workstations, their VPN endpoint) and on the business associate (cloud provider) per their BAA for their physical infrastructure.
- **BYOD:** Bring-your-own-device policies create gaps in §164.310(c) compliance because the organization controls software configuration but not physical safeguards. Separate BYOD-specific workstation security documentation is recommended.
