# CIP-015-1 — Internal Network Security Monitoring (INSM)

**Registry path:** `/regulation-registry/NERC-CIP/CIP-015-1/`
**Version:** 1 (approved 2025-06-26 by FERC Order 907; phased compliance begins 2028-10-01)
**Last parsed:** 2026-05-20
**Overall confidence:** MEDIUM — R1 (process) PARAMETERIZED; R2 (data retention) DETERMINISTIC; R3 (data protection) PARAMETERIZED

> ⚠️ **Pre-enforcement standard.** Phased compliance begins October 1, 2028 for High Impact BCS; Medium Impact with ERC follows ~24 months later. Tests below should run in informational/dark-launch mode until enforcement begins.
>
> **Scope expansion in progress.** FERC's Order 907 directed NERC to extend INSM to EACMS and PACS outside the ESP within 12 months (NERC Project 2025-02, target completion ~2026-09-01). When that expansion lands, expect a CIP-015-2 within ~2 years. The architecture below should be designed to accommodate this without rework.

---

## Scope summary

CIP-015 is the first NERC CIP standard explicitly addressing east-west traffic *inside* the Electronic Security Perimeter — historically the largest visibility gap in the suite. It applies to High Impact BES Cyber Systems and Medium Impact BES Cyber Systems with External Routable Connectivity. The standard is intentionally outcome-oriented (detect anomalous network activity; retain and protect the data) rather than prescriptive about the specific INSM mechanism, which is the main source of PARAMETERIZED elements.

The cleanest mental model: CIP-005 governs *what crosses the ESP boundary*; CIP-015 governs *what happens once traffic is inside it*.

## R1 — Implement INSM

### Source excerpt (R1)

> Each Responsible Entity shall implement one or more documented process(es) for Internal Network Security Monitoring of networks protected by the Electronic Security Perimeter(s) of high impact BES Cyber Systems and medium impact BES Cyber Systems with External Routable Connectivity to provide methods for detecting and evaluating anomalous network activity.

### Element extraction

| Element | Value | Classification |
|---|---|---|
| Subject | Networks inside the ESP of H impact BCS and M impact BCS with ERC | DETERMINISTIC |
| Condition | Asset is in scope per CIP-002 categorization AND has ERC where applicable | DETERMINISTIC |
| Obligation | (R1.1) Implement network data feed collection covering communications between Cyber Assets inside the ESP; (R1.2) Implement detection methods to identify anomalous network activity; (R1.3) Evaluate identified anomalous activity | PARAMETERIZED — "anomalous", detection method not prescribed |
| Evidence | `insm_collection_record[]`, `anomaly_detection.rules[]`, `evaluation_record.{anomaly_id, disposition, evaluated_by}` | PARAMETERIZED |

## R2 — Retain INSM Data

### Source excerpt (R2)

> Each Responsible Entity shall implement one or more documented process(es) to retain the data collected to support the detection and evaluation of anomalous network activity as identified in Requirement R1.

### Element extraction

| Element | Value | Classification |
|---|---|---|
| Subject | INSM data collected per R1 | DETERMINISTIC |
| Condition | Data was collected under R1.1 OR R1.2 detection event | DETERMINISTIC |

| Evidence | `retention_policy.{data_type, retention_days, justification}`, `retained_data_inventory[]` | PARAMETERIZED |

## R3 — Protect INSM Data

### Source excerpt (R3)

> Each Responsible Entity shall implement one or more documented process(es) to protect the data collected to support the detection and evaluation of anomalous network activity as identified in Requirement R1 from unauthorized access and unauthorized modification.

```python
"""
CIP-015-1 R1 — INSM Implementation
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
Phased enforcement: H impact 2028-10-01; M-with-ERC ~2030-10-01
"""
import pytest
from datetime import datetime, timedelta

H_ENFORCEMENT = datetime(2028, 10, 1)
M_ENFORCEMENT = datetime(2030, 10, 1)


def _in_force_for(asset_impact, has_erc):
    now = datetime.utcnow()
    if asset_impact == "High":
        return now >= H_ENFORCEMENT
    if asset_impact == "Medium" and has_erc:
        return now >= M_ENFORCEMENT
    return False


@pytest.mark.assumption(
    id="ASSUME-015-001",
    description='"Anomalous" = deny-rule match OR baseline deviation OR threat intel OR analyst flag',
    approved_by="Compliance Officer",
    review_date="2026-05-15",
)
def test_CIP015_R1_2_detection_method_implemented(esp_protected_assets):
    """R1.2: Detection method must implement at least the 4 categories
    from ASSUME-015-001."""
    required_categories = {
        "deny_rule_match",
        "baseline_deviation",
        "threat_intel_indicator",
        "analyst_flag",
    }
    for asset in esp_protected_assets:
        if not _in_force_for(asset.impact_rating, asset.has_erc):
            pytest.skip(f"CIP-015 not yet enforceable for {asset.asset_id}")
        rules = asset.insm_detection_rules
        covered = {r.category for r in rules}
        missing = required_categories - covered
        assert not missing, (
            f"VIOLATION: ESP-protected {asset.asset_id} INSM detection "
            f"missing categories: {missing} (ASSUME-015-001)."
        )


@pytest.mark.assumption(
    id="ASSUME-015-002",
    description="Coverage = documented gaps + 92-day verification + sample captures",
    approved_by="Compliance Officer",
    review_date="2026-05-15",
)
def test_CIP015_R1_1_network_coverage_complete(esp_segments):
    """R1.1: every ESP network segment must be in the INSM collection
    inventory, OR have a documented gap with mitigation."""
    for seg in esp_segments:
        if not _in_force_for(seg.impact_rating, seg.has_erc):
            pytest.skip(f"CIP-015 not yet enforceable for segment {seg.segment_id}")
        if seg.insm_collection_active:
            continue
        assert seg.collection_gap_record_id is not None, (
            f"VIOLATION (R1.1): ESP segment {seg.segment_id} has no INSM "
            f"collection and no documented gap mitigation."
        )


def test_CIP015_R1_1_coverage_verified_quarterly(insm_coverage_verifications):
    """ASSUME-015-002: verification within prior 92 days."""
    cutoff = datetime.utcnow() - timedelta(days=92)
    for v in insm_coverage_verifications:
        assert v.verified_at > cutoff, (
            f"VIOLATION: INSM coverage verification {v.verification_id} "
            f"is {(datetime.utcnow() - v.verified_at).days} days old "
            f"(threshold 92)."
        )
        assert v.sample_capture_artifact_id is not None, (
            f"VIOLATION: Verification {v.verification_id} lacks sample "
            f"capture artifact (ASSUME-015-002)."
        )


def test_CIP015_R1_3_anomalies_evaluated(anomaly_events):
    """R1.3: every detected anomaly must have an evaluation record."""
    for event in anomaly_events:
        assert event.evaluation_record_id is not None, (
            f"VIOLATION (R1.3): Anomaly {event.anomaly_id} detected "
            f"{event.detected_at} has no evaluation record."
        )
        assert event.evaluation_disposition in (
            "true_positive", "false_positive", "benign_anomaly", "escalated"
        ), (
            f"VIOLATION: Anomaly {event.anomaly_id} evaluation "
            f"disposition is '{event.evaluation_disposition}'."
        )
        assert event.evaluated_by_analyst_id is not None, (
            f"VIOLATION: Anomaly {event.anomaly_id} evaluation has no "
            f"named analyst attribution."
        )
```

```python
"""
CIP-015-1 R2 — Data Retention
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest
from datetime import datetime, timedelta


RETENTION_FLOORS = {
    "anomaly_metadata": 365,
    "analyst_case_notes": 365,
    "raw_packet_capture": 90,
    "flow_record": 90,
    "aggregated_telemetry": 180,
}


@pytest.mark.assumption(
    id="ASSUME-015-003",
    description="Retention floors: metadata/notes 365d; packets/flows 90d; aggregates 180d",
    approved_by="Compliance Officer",
    review_date="2026-05-15",
)
def test_CIP015_R2_retention_policy_meets_floor(retention_policies):
    for policy in retention_policies:
        floor = RETENTION_FLOORS.get(policy.data_type)
        if floor is None:
            continue
        assert policy.retention_days >= floor, (
            f"VIOLATION: Retention policy for '{policy.data_type}' is "
            f"{policy.retention_days} days; floor is {floor} days "
            f"(ASSUME-015-003)."
        )


def test_CIP015_R2_all_data_types_classified(retained_data_inventory):
    """Every data type in the retention storage must map to a policy."""
    classified = {item.data_type for item in retained_data_inventory if item.retention_policy_id}
    unclassified = [
        item.data_type for item in retained_data_inventory
        if not item.retention_policy_id
    ]
    assert not unclassified, (
        f"VIOLATION: INSM data types with no retention policy mapping: "
        f"{set(unclassified)}"
    )


def test_CIP015_R2_retention_actually_enforced(retained_data_inventory):
    """Items older than their policy's retention window should be either
    purged or in a documented legal-hold state."""
    now = datetime.utcnow()
    for item in retained_data_inventory:
        if not item.retention_policy:
            continue
        age = now - item.collected_at
        if age <= timedelta(days=item.retention_policy.retention_days):
            continue
        # Past retention window
        assert item.legal_hold_id is not None or item.purged, (
            f"VIOLATION: INSM data {item.item_id} is {age.days} days old "
            f"(policy: {item.retention_policy.retention_days}) and is "
            f"neither purged nor under legal hold."
        )
```

## Notes for the registry

- **Treat CIP-015 implementation as a 2-year program, not a compliance project.** The architecture decisions (where to TAP/SPAN, what data to retain, how to manage analyst access to BCSI-classified packet captures) are expensive to redo. Most entities should be building toward the H impact 2028-10-01 deadline now.
- **EACMS/PACS extension is coming.** FERC Order 907 directed NERC to expand INSM scope to EACMS/PACS outside the ESP. NERC Project 2025-02 has a target completion of ~September 2026. Design the collection architecture with extra capacity to absorb these additional segments without redesign.
- **Cross-reference to CIP-011-3:** INSM data contains BES Cyber System Information by definition (network topology, traffic patterns, authentication events). ASSUME-015-004 explicitly chains to ASSUME-011-001 — treat INSM data as BCSI. Access to it counts toward CIP-004 R4 access authorization, CIP-011 R1 handling, and CIP-011 R2 reuse/disposal.
- **Anomaly evaluation backlog is the audit hot-spot.** Detection generates anomalies faster than analysts can evaluate them. R1.3 requires every anomaly to have an evaluation record. Plan headcount accordingly, or tune detection to keep volume manageable. A months-long unevaluated queue is a finding.
- **No retention period in the standard text** — that's why ASSUME-015-003 sets floors. If the entity's policy specifies longer retention, the tests should auto-tighten via the `retention_policy.retention_days` value.
