# CIP-010-4 — Configuration Change Management and Vulnerability Assessments

**Registry path:** `/regulation-registry/NERC-CIP/CIP-010-4/`
**Version:** 4 (currently enforceable)
**Last parsed:** 2026-05-20
**Applies to:** Bulk Electric System (BES) owners, operators, and users in North America registered with NERC as a functional entity (Generator Owner/Operator, Transmission Owner/Operator, Balancing Authority, Reliability Coordinator, etc.) that own or operate BES Cyber Systems
**Trigger:** NERC registration as a functional entity + ownership/operation of BES Cyber Systems classified as High, Medium, or Low Impact per CIP-002; BES threshold: generation ≥ 20 MW, transmission ≥ 100 kV generally
**Jurisdiction:** North America — United States, Canada, and portions of Mexico; enforced by NERC and eight Regional Entities (WECC, SERC, MRO, RFC, NPCC, FRCC, TRE, SPP RE); FERC oversees NERC in the US
**Not applicable to:** Distribution-only utilities operating below the BES voltage threshold; behind-the-meter generation below 20 MW; entities not registered with NERC; natural gas pipeline operators (TSA SD-02D instead); water utilities (AWIA instead)
**Overall confidence:** MIXED — R1 (baselines) DETERMINISTIC, R2 (monitoring) DETERMINISTIC, R3 (vulnerability assessments) DETERMINISTIC for cadence but PARAMETERIZED for scope, R4 (TCA/RM) is the main PARAMETERIZED area

---

## Scope summary

CIP-010 covers configuration baselines, change management, monitoring for unauthorized changes, vulnerability assessments, and Transient Cyber Asset / Removable Media (TCA/RM) controls. R1 has the most operationally burdensome obligations because every baseline element change is a tracked event. R4 (TCA/RM) is the principal source of audit pain: the language around plans, evaluations, and reviews is intentionally implementation-agnostic.

## R1 — Configuration Change Management

| Element | Value | Classification |
|---|---|---|
| Subject | H/M Cyber Assets and associated EACMS, PACS, PCAs | DETERMINISTIC |
| Condition | Asset is in scope | DETERMINISTIC |

| Element | Value | Classification |
|---|---|---|
| Subject | H Impact Cyber Assets (and certain Medium with ERC) | DETERMINISTIC |
| Condition | Asset is in scope | DETERMINISTIC |

## R3 — Vulnerability Assessments

| Element | Value | Classification |
|---|---|---|
| Subject | H/M Cyber Assets and associated EACMS, PACS, PCAs | DETERMINISTIC |
| Condition | Elapsed time since last assessment; or new asset commissioning | DETERMINISTIC |
| Obligation | (R3.1) Paper or active assessment at least every 15 calendar months; (R3.2) For H only, active assessment at least every 36 calendar months in a test environment OR active scan in production with mitigating controls; (R3.3) Prior to commissioning new H/M assets, perform active vulnerability assessment (where technically feasible); (R3.4) Document results and an action plan for identified vulnerabilities | DETERMINISTIC for cadence; PARAMETERIZED for "active vs paper" scope |
| Evidence | `vuln_assessment.{type, completed_at, scope, findings, action_plan_id}` | DETERMINISTIC for presence; PARAMETERIZED for scope adequacy |

## R4 — Transient Cyber Asset and Removable Media

| Element | Value | Classification |
|---|---|---|
| Subject | Transient Cyber Assets and Removable Media used at H/M BCS | DETERMINISTIC (defined terms) |
| Condition | TCA/RM connected to or used on H/M BCS | DETERMINISTIC |

## Generated tests

### `tests/CIP-010/test_R1.py` (Pattern 1)

```python
"""
CIP-010-4 R1 — Configuration Change Management
Confidence: HIGH | Human Review: NOT REQUIRED
"""
from datetime import timedelta

REQUIRED_BASELINE_ELEMENTS = {
    "os_or_firmware",
    "commercial_or_oss_apps",
    "custom_software",
    "logical_network_accessible_ports",
    "security_patches_applied",
}


def test_CIP010_R1_1_baseline_completeness(in_scope_assets):
    for asset in in_scope_assets:
        baseline = asset.baseline
        assert baseline is not None, (
            f"CRITICAL VIOLATION: {asset.asset_id} has no baseline."
        )
        missing = REQUIRED_BASELINE_ELEMENTS - set(baseline.elements_captured)
        assert not missing, (
            f"VIOLATION: {asset.asset_id} baseline missing elements: {missing}"
        )


def test_CIP010_R1_2_changes_authorized(change_records):
    for change in change_records:
        assert change.authorized_by is not None, (
            f"VIOLATION: Change {change.change_id} on {change.asset_id} "
            f"applied with no authorization record."
        )


def test_CIP010_R1_3_baseline_updated_within_30_days(change_records):
    for change in change_records:
        if change.baseline_updated_at is None:
            assert False, (
                f"VIOLATION: Change {change.change_id} has no baseline "
                f"update timestamp."
            )
        delta = change.baseline_updated_at - change.applied_at
        assert delta <= timedelta(days=30), (
            f"VIOLATION: Change {change.change_id} baseline updated "
            f"{delta.days} days after application (threshold 30)."
        )
```

"""
CIP-010-4 R4 — Transient Cyber Asset and Removable Media
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest
from datetime import datetime, timedelta


@pytest.mark.assumption(
    id="ASSUME-010-001",
    description="TCA pre-connect = AV sigs <7d + full scan <24h + clean/quarantined",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP010_R4_tca_pre_connection_scan(tca_use_events):
    for event in tca_use_events:
        assert event.av_signatures_age_days <= 7, (
            f"VIOLATION: TCA {event.tca_id} connected with AV signatures "
            f"{event.av_signatures_age_days} days old (ASSUME-010-001)."
        )
        scan_age = event.connection_at - event.last_full_scan_at
        assert scan_age <= timedelta(hours=24), (
            f"VIOLATION: TCA {event.tca_id} last full scan "
            f"{scan_age.total_seconds() / 3600:.1f}h before connection."
        )
        assert event.scan_result in ("clean", "quarantined"), (
            f"VIOLATION: TCA {event.tca_id} connected with scan result "
            f"'{event.scan_result}'."
        )


def test_CIP010_R4_removable_media_scanned(rm_use_events):
    for event in rm_use_events:
        assert event.scanned_for_malicious_code_at is not None, (
            f"VIOLATION: Removable media {event.rm_id} used on "
            f"{event.target_asset_id} without a scan record."
        )

## Notes for the registry

- **R1.3's 30-day baseline update window is the most common drift point.** Engineering teams update production but leave baselines stale, then audit-prep races to backfill. Recommend automating baseline regeneration on change ticket close.
- **R1.4 / R1.5 verifications are conditional.** R1.4 (security control verification) applies only to H/M; R1.5 (test environment) applies only to H. Test fixtures should select asset scope before applying these tests.
- **R4 Attachment 1 sections** include managed vs unmanaged TCAs, removable media, ongoing vs on-demand reviews. The Pattern 2 approach surfaces the pre-connect scan as the verifiable artifact; the plan's substantive review remains manual.
