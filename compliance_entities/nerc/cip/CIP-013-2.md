# CIP-013-2 — Supply Chain Risk Management

**Registry path:** `/regulation-registry/NERC-CIP/CIP-013-2/`
**Version:** 2 (currently enforceable since 2022-10-01)
**Last parsed:** 2026-05-20
**Applies to:** Bulk Electric System (BES) owners, operators, and users in North America registered with NERC as a functional entity (Generator Owner/Operator, Transmission Owner/Operator, Balancing Authority, Reliability Coordinator, etc.) that own or operate BES Cyber Systems
**Trigger:** NERC registration as a functional entity + ownership/operation of BES Cyber Systems classified as High, Medium, or Low Impact per CIP-002; BES threshold: generation ≥ 20 MW, transmission ≥ 100 kV generally
**Jurisdiction:** North America — United States, Canada, and portions of Mexico; enforced by NERC and eight Regional Entities (WECC, SERC, MRO, RFC, NPCC, FRCC, TRE, SPP RE); FERC oversees NERC in the US
**Not applicable to:** Distribution-only utilities operating below the BES voltage threshold; behind-the-meter generation below 20 MW; entities not registered with NERC; natural gas pipeline operators (TSA SD-02D instead); water utilities (AWIA instead)
**Overall confidence:** MIXED — R1 (plan content) PARAMETERIZED, R2 (plan implementation) PARAMETERIZED, R3 (plan review) DETERMINISTIC

---

## Scope summary

CIP-013 governs the supply chain risk management plans for procuring high and medium impact BES Cyber Systems and their associated EACMS and PACS (v2 added EACMS/PACS to scope). Failures here tend to be of the "plan exists but isn't followed during the actual procurement" variety — the plan-vs-implementation gap.

This standard pairs tightly with:
- **CIP-005-7 R2/R3** (vendor remote access — operational counterpart to procurement-time supply chain risk)
- **CIP-010-4 R1.6** (verifying software integrity and authenticity prior to baseline change)
- **CIP-003-9 §6** (Low Impact vendor remote access)

## R1 — Supply Chain Cyber Security Risk Management Plan

### Source excerpt (R1)

> Each Responsible Entity shall develop one or more documented supply chain cyber security risk management plan(s) for high and medium impact BES Cyber Systems and their associated EACMS and PACS. The plan(s) shall include: (R1.1) process to identify and assess cyber security risk(s) from procuring and installing vendor equipment and software, and from transitions from one vendor to another; (R1.2) process to address procurements addressing six specified topics (notification of vendor security incidents, coordinated incident response, vendor-disclosure of known vulnerabilities, software integrity verification, coordinated vendor remote access controls, disclosure of vendor remote access by vendors).

| Evidence | `scrm_plan.{sections_covered, topics_covered, approved_at, approved_by}` | DETERMINISTIC for presence |

### R1.2's six required topics

1. Notification by vendor of vendor-identified incidents related to products/services
2. Coordination of vendor incident response
3. Notification by vendor when vendor remote or onsite access is no longer required
4. Disclosure by vendor of known vulnerabilities related to products/services
5. Verification of software integrity and authenticity before installation
6. Coordination of vendor-initiated Interactive Remote Access and system-to-system remote access

## Generated tests

### `tests/CIP-013/test_R1.py` (Pattern 2)

```python
"""
CIP-013-2 R1 — Supply Chain Risk Management Plan
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest

REQUIRED_R1_2_TOPICS = {
    "vendor_security_incident_notification",
    "vendor_incident_response_coordination",
    "vendor_remote_access_no_longer_required_notification",
    "vendor_vulnerability_disclosure",
    "software_integrity_authenticity_verification",
    "vendor_remote_access_coordination",
}


@pytest.mark.assumption(
    id="ASSUME-013-001",
    description="A topic is 'addressed' = named section + procurement artifact + 15mo review",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP013_R1_1_risk_assessment_process_documented(scrm_plans):
    for plan in scrm_plans:
        assert plan.has_risk_identification_process, (
            f"VIOLATION (R1.1): SCRM plan {plan.plan_id} lacks a documented "
            f"risk identification and assessment process."
        )
        assert plan.has_vendor_transition_process, (
            f"VIOLATION (R1.1): SCRM plan {plan.plan_id} does not address "
            f"transitions from one vendor to another."
        )


def test_CIP013_R1_2_six_topics_covered(scrm_plans):
    for plan in scrm_plans:
        missing = REQUIRED_R1_2_TOPICS - set(plan.r1_2_topics_covered)
        assert not missing, (
            f"VIOLATION (R1.2): SCRM plan {plan.plan_id} missing required "
            f"procurement topics: {missing}"
        )


def test_CIP013_R1_2_topics_have_artifacts(scrm_plans):
    """Each R1.2 topic must reference a concrete artifact (contract clause
    template, RFP language, or vendor attestation template) per ASSUME-013-001."""
    for plan in scrm_plans:
        for topic in REQUIRED_R1_2_TOPICS:
            artifact = plan.artifact_for_topic.get(topic)
            assert artifact is not None, (
                f"VIOLATION: SCRM plan {plan.plan_id} addresses topic "
                f"'{topic}' but references no concrete artifact "
                f"(ASSUME-013-001)."
            )
```

### `tests/CIP-013/test_R2.py` (Pattern 2)

```python
"""
CIP-013-2 R2 — Plan Implementation
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest


@pytest.mark.assumption(
    id="ASSUME-013-002",
    description="In-scope = installed in H/M BCS/EACMS/PACS OR exceeds threshold OR is software",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP013_R2_risk_assessment_before_award(procurements):
    for proc in procurements:
        if not proc.in_scope:
            continue
        assert proc.risk_assessment_id is not None, (
            f"VIOLATION: In-scope procurement {proc.procurement_id} for "
            f"{proc.vendor_name} awarded with no risk assessment "
            f"(ASSUME-013-002)."
        )
        assert proc.risk_assessment_completed_at <= proc.contract_award_at, (
            f"VIOLATION: Procurement {proc.procurement_id} risk assessment "
            f"completed after contract award."
        )


def test_CIP013_R2_software_integrity_check(procurements):
    """R1.2 topic 5: verify software integrity and authenticity before install."""
    for proc in procurements:
        if not proc.in_scope or not proc.includes_software:
            continue
        assert proc.software_integrity_check_id is not None, (
            f"VIOLATION: Procurement {proc.procurement_id} includes software "
            f"but has no integrity/authenticity verification record."
        )
        if proc.installation_completed_at:
            assert proc.software_integrity_check_at <= proc.installation_completed_at, (
                f"VIOLATION: Software for procurement {proc.procurement_id} "
                f"was installed before integrity check."
            )


def test_CIP013_R2_vendor_attestations_present(procurements):
    """Each in-scope procurement must have attestations for each R1.2 topic
    that applies to it."""
    for proc in procurements:
        if not proc.in_scope:
            continue
        missing = set(proc.required_attestation_topics) - \
                  set(proc.vendor_attestation_topics_received)
        assert not missing, (
            f"VIOLATION: Procurement {proc.procurement_id} from "
            f"{proc.vendor_name} missing vendor attestations for: {missing}"
        )
```

from datetime import datetime, timedelta

REVIEW_WINDOW_DAYS = 457  # 15 calendar months


def test_CIP013_R3_review_cadence(scrm_plans):
    now = datetime.utcnow()
    for plan in scrm_plans:
        age = now - plan.last_reviewed_at
        assert age <= timedelta(days=REVIEW_WINDOW_DAYS), (
            f"VIOLATION: SCRM plan {plan.plan_id} last reviewed "
            f"{age.days} days ago (threshold {REVIEW_WINDOW_DAYS})."

            f"INTEGRITY FAILURE: SCRM plan {plan.plan_id} lacks signature."

## Notes for the registry

- **The plan-vs-implementation gap is the audit hot-spot.** R1 tests are easy to pass (plan documents exist). R2 tests are where real entities fail — because procurement happens in a different system than compliance, and the SCRM checklist often isn't enforced at award gates. Recommend integrating the SCRM checklist as a hard gate in the procurement workflow rather than an after-the-fact compliance check.
- **v2 EACMS/PACS expansion:** Many entities still scope CIP-013 to BCS only out of habit. Make sure procurement classification logic tags EACMS and PACS items correctly.
- **Software integrity verification** (R1.2 topic 5) overlaps with CIP-010-4 R1.6. CIP-013 covers the contractual/process requirement; CIP-010 covers the technical verification at baseline-change time. Both must be evidenced.
- **Cross-reference to CIP-005-7 R3 and CIP-003-9 §6:** the vendor remote access concerns in R1.2 topics 3 and 6 are realized operationally in those standards. Treat CIP-013 as the procurement-time control and CIP-005/CIP-003 as the runtime control.
