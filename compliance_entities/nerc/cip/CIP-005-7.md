## Scope summary

CIP-005 defines the Electronic Security Perimeter (ESP) and governs all routable traffic crossing it. The v7 revision (2023) added explicit vendor remote access controls for High and Medium Impact systems — paralleling the Low Impact additions in CIP-003-9. Most ESP requirements are technically deterministic (deny-by-default, only permitted ports/protocols) but their evidence is operational logs whose completeness must be validated.

"""
CIP-005-7 R1 — Electronic Security Perimeter
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED for R1.3
"""
import pytest


def test_CIP005_R1_1_routable_assets_in_esp(hm_cyber_assets):
    """R1.1: Routable-connected H/M assets must be inside an ESP."""
    for asset in hm_cyber_assets:
        if asset.is_routable:
            assert asset.esp_membership_id is not None, (
                f"CRITICAL VIOLATION: {asset.asset_id} is routable but "
                f"has no ESP membership."
            )


def test_CIP005_R1_default_deny(firewalls_at_esp_boundary):
    """Default-deny posture at every ESP boundary device."""
    for fw in firewalls_at_esp_boundary:
        assert fw.default_action.upper() == "DENY", (
            f"CRITICAL VIOLATION: {fw.device_id} default action is "
            f"{fw.default_action}, not DENY."
        )


@pytest.mark.assumption(
    id="ASSUME-005-001",
    description='"Needed" = documented justification + named owner + 15mo review',
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP005_R1_3_permitted_rules_justified(esp_firewall_rules):
    """R1.3: All permit rules must satisfy the entity's 'needed' criteria."""
    from datetime import datetime, timedelta
    cutoff = datetime.utcnow() - timedelta(days=457)
    for rule in esp_firewall_rules:
        if rule.action.upper() != "PERMIT":
            continue
        assert rule.justification, (
            f"VIOLATION: Rule {rule.rule_id} on {rule.device_id} "
            f"permits traffic with no documented justification "
            f"(ASSUME-005-001)."
        )
        assert rule.owner_id, (
            f"VIOLATION: Rule {rule.rule_id} has no named owner."
        )
        assert rule.last_reviewed_at > cutoff, (
            f"STALE: Rule {rule.rule_id} last reviewed "
            f"{rule.last_reviewed_at}. Threshold: 15 months."
        )

## Notes for the registry

- **TFE (Technical Feasibility Exception):** R1.4 dial-up authentication may legitimately be unable to satisfy if equipment is old enough. The TFE process produces a discrete artifact that can be tested for presence — but the substance is a CONTESTED determination filed with the Regional Entity. Add a Pattern 3 test for TFE currency.
- **Pairing with CIP-003-9 Section 6:** Both standards now address vendor remote access — R3 covers H/M, CIP-003-9 covers Low. Avoid duplicate evidence collection by sharing the underlying vendor session log.
