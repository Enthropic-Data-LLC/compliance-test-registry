# CIP-011-3 — Information Protection

**Registry path:** `/regulation-registry/NERC-CIP/CIP-011-3/`
**Version:** 3 (currently enforceable)
**Last parsed:** 2026-05-20
**Applies to:** Bulk Electric System (BES) owners, operators, and users in North America registered with NERC as a functional entity (Generator Owner/Operator, Transmission Owner/Operator, Balancing Authority, Reliability Coordinator, etc.) that own or operate BES Cyber Systems
**Trigger:** NERC registration as a functional entity + ownership/operation of BES Cyber Systems classified as High, Medium, or Low Impact per CIP-002; BES threshold: generation ≥ 20 MW, transmission ≥ 100 kV generally
**Jurisdiction:** North America — United States, Canada, and portions of Mexico; enforced by NERC and eight Regional Entities (WECC, SERC, MRO, RFC, NPCC, FRCC, TRE, SPP RE); FERC oversees NERC in the US
**Not applicable to:** Distribution-only utilities operating below the BES voltage threshold; behind-the-meter generation below 20 MW; entities not registered with NERC; natural gas pipeline operators (TSA SD-02D instead); water utilities (AWIA instead)
**Overall confidence:** MIXED — R1 (identification/protection of BCSI) PARAMETERIZED, R2 (reuse and disposal) DETERMINISTIC

---

## Scope summary

CIP-011 protects BES Cyber System Information (BCSI). Version 3 broadened the standard to address BCSI held by third parties (cloud, contractors) and introduced explicit handling for access provisioning. Most failures come from inconsistent BCSI identification rather than control failures — the test surface focuses on classification coverage and disposal traceability.

---

## R1 — Information Protection

| Element | Value | Classification |
|---|---|---|
| Subject | BES Cyber System Information (BCSI) | DETERMINISTIC (defined term) but PARAMETERIZED in practice — entity must identify what qualifies |
| Condition | Information meets BCSI definition | PARAMETERIZED — classification judgment |
| Obligation | (R1.1) Identify and classify BCSI; (R1.2) Procedure(s) for protecting and securely handling BCSI (storage, transit, use); access provisioning for BCSI per CIP-004 | PARAMETERIZED — "protect and securely handle" |
| Evidence | `bcsi_inventory[]`, `bcsi_handling_procedure`, `bcsi_access_provisioning_record` | PARAMETERIZED |

## R2 — BES Cyber Asset Reuse and Disposal

| Element | Value | Classification |
|---|---|---|
| Subject | BES Cyber Assets (and PCAs, EACMS, PACS) being released for reuse or disposal | DETERMINISTIC |
| Condition | Asset being reused outside the original BCS OR disposed of | DETERMINISTIC |
| Obligation | (R2.1) Prior to release for reuse outside the original BCS, take action to prevent the unauthorized retrieval of BCSI from the data storage media; (R2.2) Prior to disposal, take action to prevent unauthorized retrieval of BCSI from the data storage media | DETERMINISTIC for trigger; PARAMETERIZED for "take action" method |
| Evidence | `disposal_record.{action_type, performed_at, performed_by, asset_serial, verification_method}` | DETERMINISTIC for presence |

### `tests/CIP-011/test_R1.py` (Pattern 2)

```python
"""
CIP-011-3 R1 — Information Protection
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest
from datetime import datetime, timedelta


PRESUMED_BCSI_CATEGORIES = {
    "esp_network_diagram",
    "asset_baseline",
    "vulnerability_assessment_result",
    "incident_response_detail",
    "credential_store_metadata",
}


@pytest.mark.assumption(
    id="ASSUME-011-001",
    description="BCSI is presumed for 5 named categories; borderline = 30-day named review",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP011_R1_1_bcsi_identification_coverage(document_inventory):
    """All presumed BCSI categories must be classified as BCSI."""
    for doc in document_inventory:
        if doc.category in PRESUMED_BCSI_CATEGORIES:
            assert doc.bcsi_classification == "BCSI", (
                f"VIOLATION: Document {doc.doc_id} category={doc.category} "
                f"is presumed BCSI but classified as "
                f"'{doc.bcsi_classification}' (ASSUME-011-001)."
            )


def test_CIP011_R1_borderline_classified_within_30_days(borderline_documents):
    cutoff = datetime.utcnow() - timedelta(days=30)
    for doc in borderline_documents:
        if doc.received_at < cutoff:
            assert doc.bcsi_classification in ("BCSI", "Not BCSI"), (
                f"VIOLATION: Borderline document {doc.doc_id} received "
                f"{doc.received_at} has no classification after 30 days."
            )
            assert doc.classified_by is not None, (
                f"VIOLATION: Document {doc.doc_id} classification has no "
                f"named reviewer."
            )


@pytest.mark.assumption(
    id="ASSUME-011-002",
    description="Protect BCSI = encrypt at rest + TLS in transit + access logging",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP011_R1_2_bcsi_protection_controls(bcsi_storage_systems):
    for system in bcsi_storage_systems:
        assert system.at_rest_encryption_enabled, (
            f"VIOLATION: BCSI store {system.system_id} lacks at-rest encryption."
        )
        assert system.kms_managed, (
            f"VIOLATION: BCSI store {system.system_id} encryption keys not "
            f"under KMS management."
        )
        assert system.in_transit_protocol in ("TLS1.2", "TLS1.3"), (
            f"VIOLATION: BCSI store {system.system_id} in-transit protocol "
            f"is {system.in_transit_protocol}."
        )
        assert system.access_logging_enabled, (
            f"VIOLATION: BCSI store {system.system_id} has no access logging."
        )
```

### `tests/CIP-011/test_R2.py` (Pattern 2)

```python
"""
CIP-011-3 R2 — BES Cyber Asset Reuse and Disposal
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest


APPROVED_SANITIZATION_METHODS = {
    "nist_800_88_clear",
    "nist_800_88_purge",
    "physical_destruction",
    "cryptographic_erase",
}


@pytest.mark.assumption(
    id="ASSUME-011-003",
    description="Sanitization = NIST 800-88 Clear/Purge OR physical destruction OR crypto-erase",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP011_R2_sanitization_method(disposal_records):
    for record in disposal_records:
        assert record.action_type in APPROVED_SANITIZATION_METHODS, (
            f"VIOLATION: Disposal {record.record_id} for asset "
            f"{record.asset_serial} used method '{record.action_type}', "
            f"not in approved list (ASSUME-011-003)."
        )


def test_CIP011_R2_record_completeness(disposal_records):
    required = {"action_type", "performed_at", "performed_by",
                "asset_serial", "verification_method"}
    for record in disposal_records:
        missing = required - set(record.fields_present)
        assert not missing, (
            f"VIOLATION: Disposal {record.record_id} missing required "
            f"fields: {missing}"
        )


def test_CIP011_R2_chain_of_custody_for_destruction(disposal_records):
    for record in disposal_records:
        if record.action_type != "physical_destruction":
            continue
        assert record.chain_of_custody_log_id is not None, (
            f"VIOLATION: Physical destruction record {record.record_id} "
            f"has no chain-of-custody log reference."
        )
```

## Notes for the registry

- **R1 is heavily PARAMETERIZED** because BCSI identification is the entity's own determination. The framework's approach is to enumerate presumed-BCSI categories (ASSUME-011-001), test that those are always classified, and route borderline cases through a 30-day named-reviewer process.
- **R2 audit pitfall:** Field-deployed assets disposed by site personnel (e.g., aging serial-to-Ethernet converters) often skip the formal sanitization workflow. The test catches presence of records; it does not catch assets retired without a record being created — that requires a complementary inventory-reconciliation test outside CIP-011's scope.
- **v3 third-party BCSI:** Cloud storage and contractor handling of BCSI is now explicitly in scope. Make sure `bcsi_storage_systems` fixtures include third-party-hosted stores.
