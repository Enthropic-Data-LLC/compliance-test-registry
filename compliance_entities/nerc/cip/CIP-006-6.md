# CIP-006-6 — Physical Security of BES Cyber Systems

**Registry path:** `/regulation-registry/NERC-CIP/CIP-006-6/`
**Version:** 6 (currently enforceable)
**Last parsed:** 2026-05-20
**Applies to:** Bulk Electric System (BES) owners, operators, and users in North America registered with NERC as a functional entity (Generator Owner/Operator, Transmission Owner/Operator, Balancing Authority, Reliability Coordinator, etc.) that own or operate BES Cyber Systems
**Trigger:** NERC registration as a functional entity + ownership/operation of BES Cyber Systems classified as High, Medium, or Low Impact per CIP-002; BES threshold: generation ≥ 20 MW, transmission ≥ 100 kV generally
**Jurisdiction:** North America — United States, Canada, and portions of Mexico; enforced by NERC and eight Regional Entities (WECC, SERC, MRO, RFC, NPCC, FRCC, TRE, SPP RE); FERC oversees NERC in the US
**Not applicable to:** Distribution-only utilities operating below the BES voltage threshold; behind-the-meter generation below 20 MW; entities not registered with NERC; natural gas pipeline operators (TSA SD-02D instead); water utilities (AWIA instead)
**Overall confidence:** HIGH — most requirements are presence-and-cadence assertions with crisp time windows

---

## Scope summary

CIP-006 establishes the Physical Security Perimeter (PSP) concept and the operational controls inside it: access authorization (handed off to CIP-004), monitoring, logging, escorting, and maintenance/testing of physical access control systems (PACS). The standard is unusually friendly to automation because nearly every obligation has a numeric threshold (24 hours, 15 minutes, 7 calendar days, 24 calendar months) attached.

"""
CIP-006-6 R1 — Physical Security Plan
Confidence: HIGH | Human Review: NOT REQUIRED
"""
from datetime import datetime, timedelta

LOG_RETENTION_DAYS = 90


def test_CIP006_R1_psp_defined_for_hm_assets(hm_cyber_assets):
    for asset in hm_cyber_assets:
        assert asset.psp_id is not None, (
            f"CRITICAL VIOLATION: {asset.asset_id} is H/M but has "
            f"no PSP membership."
        )


def test_CIP006_R1_entry_points_monitored(psp_entry_points):
    for ep in psp_entry_points:
        assert ep.monitoring_method in ("alarm", "human_observation", "both"), (
            f"VIOLATION: Entry point {ep.entry_point_id} on PSP "
            f"{ep.psp_id} has monitoring_method='{ep.monitoring_method}'."
        )


def test_CIP006_R1_log_retention(physical_access_logs):
    cutoff = datetime.utcnow() - timedelta(days=LOG_RETENTION_DAYS)
    for log in physical_access_logs:
        # The log's earliest entry should be at least 90 days old OR
        # the log should not yet have rolled off — i.e. its earliest
        # entry should not be inside the retention window if older
        # entries existed.
        assert log.earliest_entry_at <= cutoff or log.has_full_history, (
            f"VIOLATION: PSP {log.psp_id} log appears to be "
            f"truncated inside the 90-day retention window."
        )


def test_CIP006_R1_unauthorized_access_responded(unauthorized_access_alarms):
    for alarm in unauthorized_access_alarms:
        assert alarm.response_recorded_at is not None, (
            f"VIOLATION: Alarm {alarm.alarm_id} at PSP {alarm.psp_id} "
            f"({alarm.detected_at}) has no recorded response."
        )
```

"""
CIP-006-6 R1.10 — ESP-Internal Cabling Outside PSP
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest


ACCEPTABLE_PROTECTIONS = {"physical_restriction", "encryption", "monitored_link"}


@pytest.mark.assumption(
    id="ASSUME-006-001",
    description='"Equally effective" = continuous link-state + tamper detection',
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP006_R1_10_cabling_protected(esp_internal_cable_runs):
    for run in esp_internal_cable_runs:
        if not run.outside_psp:
            continue
        if run.protection_method in ACCEPTABLE_PROTECTIONS:
            continue
        # Fall back to ASSUME-006-001 definition for "equally effective"
        assert run.has_link_state_monitoring and run.has_tamper_detection, (
            f"VIOLATION: Cable run {run.run_id} between {run.endpoint_a} "
            f"and {run.endpoint_b} is outside a PSP and does not satisfy "
            f"any of {ACCEPTABLE_PROTECTIONS} or ASSUME-006-001 criteria."
        )
```

        assert e.escort_id is not None, (

        f"VIOLATION: {len(rolled_off)} visitor entries purged before "
        f"90-day retention: {[e.entry_id for e in rolled_off[:5]]}..."

"""
CIP-006-6 R3 — PACS Maintenance & Testing
Confidence: HIGH | Human Review: NOT REQUIRED
"""
from datetime import datetime, timedelta

PACS_TEST_WINDOW_DAYS = 731  # 24 calendar months


def test_CIP006_R3_pacs_test_cadence(pacs_devices):
    now = datetime.utcnow()
    for device in pacs_devices:
        assert device.last_tested_at is not None, (
            f"CRITICAL VIOLATION: PACS {device.device_id} has no "
            f"recorded test history."
        )
        age = now - device.last_tested_at
        assert age <= timedelta(days=PACS_TEST_WINDOW_DAYS), (
            f"VIOLATION: PACS {device.device_id} last tested "
            f"{age.days} days ago (threshold {PACS_TEST_WINDOW_DAYS})."
        )


def test_CIP006_R3_deficiencies_remediated(pacs_test_records):
    for record in pacs_test_records:
        if record.test_result == "PASS":
            continue
        assert record.deficiencies_remediated_at is not None, (
            f"VIOLATION: PACS test {record.record_id} on "
            f"{record.device_id} found deficiencies but no "
            f"remediation recorded."
        )

## Notes for the registry

- **R1.10 is the only PARAMETERIZED carve-out.** Everything else here is presence + cadence.
- **Coordination with CIP-014:** CIP-006 covers BES Cyber System physical security; CIP-014 covers transmission station physical security. They overlap at the station boundary but address different threat models.
- **Audit hot-spot:** PACS test currency. The 24-month clock is generous, but utilities with hundreds of devices commonly let a handful slip into noncompliance without proactive scheduling.
