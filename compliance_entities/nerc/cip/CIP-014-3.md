# CIP-014-3 — Physical Security

**Registry path:** `/regulation-registry/NERC-CIP/CIP-014-3/`
**Version:** 3 (currently enforceable)
**Last parsed:** 2026-05-20
**Applies to:** Bulk Electric System (BES) owners, operators, and users in North America registered with NERC as a functional entity (Generator Owner/Operator, Transmission Owner/Operator, Balancing Authority, Reliability Coordinator, etc.) that own or operate BES Cyber Systems
**Trigger:** NERC registration as a functional entity + ownership/operation of BES Cyber Systems classified as High, Medium, or Low Impact per CIP-002; BES threshold: generation ≥ 20 MW, transmission ≥ 100 kV generally
**Jurisdiction:** North America — United States, Canada, and portions of Mexico; enforced by NERC and eight Regional Entities (WECC, SERC, MRO, RFC, NPCC, FRCC, TRE, SPP RE); FERC oversees NERC in the US
**Not applicable to:** Distribution-only utilities operating below the BES voltage threshold; behind-the-meter generation below 20 MW; entities not registered with NERC; natural gas pipeline operators (TSA SD-02D instead); water utilities (AWIA instead)
**Overall confidence:** LOW for automation — this is the most human-judgment-heavy CIP standard. R1, R2, R4, R5 are largely CONTESTED/PARAMETERIZED. R3, R6 are DETERMINISTIC (cadence/verification cycle).

---

## Scope summary

CIP-014 is structurally different from the rest of the CIP suite: it applies to *Transmission stations and substations* (not BES Cyber Systems) and the *Control Centers that operationally control them*, based on whether physical loss could result in "instability, uncontrolled separation, or Cascading within an Interconnection." Applicability is determined by a risk assessment, not by an impact-rating bright-line, which makes the entire standard PARAMETERIZED at the scoping step.

This is the standard most resistant to the RDF approach. The test surface is mostly "presence-and-currency of human determinations" rather than "did the system do the thing." Most failures here are flagged via Pattern 3 (human-surfacing) tests, not Pattern 1.

### `tests/CIP-014/test_R1.py` (Pattern 2)

```python
"""
CIP-014-3 R1 — Risk Assessment Cadence
Confidence: MEDIUM | Human Review: ASSUMPTION REQUIRED
"""
import pytest
from datetime import datetime, timedelta

TO_CADENCE_DAYS = 30 * 30.44   # 30 calendar months ≈ 913 days, use conservative
TOP_CADENCE_DAYS = 60 * 30.44  # 60 calendar months ≈ 1827 days, use conservative


@pytest.mark.assumption(
    id="ASSUME-014-001",
    description="Methodology acceptable = documented + TPL-001-consistent + station-loss-as-contingency + 12mo reviewed",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP014_R1_assessment_cadence_TO(transmission_owner_records):
    now = datetime.utcnow()
    for record in transmission_owner_records:
        latest = record.latest_risk_assessment_at
        assert latest is not None, (
            f"CRITICAL VIOLATION: TO {record.entity_id} has no R1 "
            f"risk assessment on record."
        )
        age = now - latest
        assert age <= timedelta(days=int(TO_CADENCE_DAYS)), (
            f"VIOLATION: TO {record.entity_id} R1 assessment is "
            f"{age.days} days old (threshold {int(TO_CADENCE_DAYS)} days "
            f"= 30 calendar months)."
        )


def test_CIP014_R1_assessment_cadence_TOP(transmission_operator_records):
    """Transmission Operators with applicable CCs have a 60-month clock."""
    now = datetime.utcnow()
    for record in transmission_operator_records:
        if not record.has_applicable_control_center:
            continue
        latest = record.latest_risk_assessment_at
        assert latest is not None, (
            f"CRITICAL VIOLATION: TOP {record.entity_id} has applicable "
            f"CC but no R1 risk assessment."
        )
        age = now - latest
        assert age <= timedelta(days=int(TOP_CADENCE_DAYS)), (
            f"VIOLATION: TOP {record.entity_id} R1 assessment is "
            f"{age.days} days old (threshold {int(TOP_CADENCE_DAYS)} days "
            f"= 60 calendar months)."
        )


def test_CIP014_R1_methodology_documented(risk_assessments):
    for ra in risk_assessments:
        assert ra.methodology_id is not None, (
            f"VIOLATION: R1 assessment {ra.assessment_id} has no "
            f"documented methodology reference (ASSUME-014-001)."
        )
        assert ra.compliance_review_at and \
               ra.compliance_review_at > datetime.utcnow() - timedelta(days=365), (
            f"STALE: R1 assessment {ra.assessment_id} methodology has "
            f"not been Compliance-reviewed within 12 months."
        )
```

"""
CIP-014-3 R5 — Physical Security Plan
Confidence: LOW (CONTESTED on measure adequacy)
Human Review: MANDATORY
"""
import pytest

REQUIRED_FUNCTIONS = {"deter", "detect", "delay", "assess", "communicate", "respond"}


@pytest.mark.human_review_required
@pytest.mark.contested(
    regulation="CIP-014-R5",
    reason="Substantive adequacy of physical security measures is a Compliance Officer determination",
)
def test_CIP014_R5_plan_exists_per_station(identified_stations):
    """This test verifies plan presence. Adequacy is human-determined."""
    for station in identified_stations:
        assert station.physical_security_plan_id is not None, (
            f"CRITICAL VIOLATION: Identified station {station.station_id} "
            f"has no physical security plan."
        )


@pytest.mark.assumption(
    id="ASSUME-014-003",
    description="Six required functions covered = explicit function-to-measure matrix + 12mo review",
    approved_by="Compliance Officer",
    review_date="2026-01-15",
)
def test_CIP014_R5_six_functions_addressed(physical_security_plans):
    """Verify the plan has a function-to-measure mapping for each of the
    six required functions. Does NOT verify measure adequacy."""
    for plan in physical_security_plans:
        for fn in REQUIRED_FUNCTIONS:
            measures = plan.function_to_measure_matrix.get(fn, [])
            assert measures, (
                f"VIOLATION: Plan {plan.plan_id} for station "
                f"{plan.station_id} does not map any measure to "
                f"function '{fn}' (ASSUME-014-003)."
            )


@pytest.mark.human_review_required
@pytest.mark.contested(
    regulation="CIP-014-R5",
    reason="'Designed collectively' adequacy is a Compliance Officer determination",
)
def test_CIP014_R5_compliance_officer_attestation_current(physical_security_plans):
    """Plan must carry a Compliance Officer attestation that the
    collective measures are adequate, refreshed within 12 months."""
    from datetime import datetime, timedelta
    cutoff = datetime.utcnow() - timedelta(days=365)
    for plan in physical_security_plans:
        att = plan.compliance_officer_attestation
        assert att is not None, (
            f"VIOLATION: Plan {plan.plan_id} lacks Compliance Officer "
            f"attestation of collective measure adequacy."
        )
        assert att.signed_at > cutoff, (
            f"STALE: Plan {plan.plan_id} Compliance Officer attestation "
            f"signed {att.signed_at}; refresh required."
        )

## Notes for the registry

- **CIP-014 is the standard most resistant to automation.** R1, R4, R5 all hinge on engineering judgment that auditors evaluate substantively. The framework's response is Pattern 3 human-surfacing tests that verify the presence and currency of named human determinations.
- **R5 is the canonical CONTESTED case** for physical security. The "deter, detect, delay, assess, communicate, respond" formulation is from DoD/DHS physical security doctrine but is not given quantitative thresholds. The test verifies the function-to-measure matrix exists; substantive adequacy is escalated.
- **Audit hot-spot:** R6 recommendation responses. Many entities receive third-party recommendations and never formally respond. Each recommendation needs a documented accept/reject/defer with rationale or timeline.
- **No bright-line scoping:** unlike CIP-002, CIP-014's scoping IS the entity's risk assessment. A station that fails the criteria in one assessment cycle may pass in the next due to grid topology changes. Maintain a delta-log between cycles for audit defensibility.
