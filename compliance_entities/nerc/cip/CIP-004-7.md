# CIP-004-7 — Personnel & Training

**Registry path:** `/regulation-registry/NERC-CIP/CIP-004-7/`
**Version:** 7 (currently enforceable)
**Last parsed:** 2026-05-20
**Applies to:** Bulk Electric System (BES) owners, operators, and users in North America registered with NERC as a functional entity (Generator Owner/Operator, Transmission Owner/Operator, Balancing Authority, Reliability Coordinator, etc.) that own or operate BES Cyber Systems
**Trigger:** NERC registration as a functional entity + ownership/operation of BES Cyber Systems classified as High, Medium, or Low Impact per CIP-002; BES threshold: generation ≥ 20 MW, transmission ≥ 100 kV generally
**Jurisdiction:** North America — United States, Canada, and portions of Mexico; enforced by NERC and eight Regional Entities (WECC, SERC, MRO, RFC, NPCC, FRCC, TRE, SPP RE); FERC oversees NERC in the US
**Not applicable to:** Distribution-only utilities operating below the BES voltage threshold; behind-the-meter generation below 20 MW; entities not registered with NERC; natural gas pipeline operators (TSA SD-02D instead); water utilities (AWIA instead)
**Overall confidence:** HIGH for R1–R3 and R5, MEDIUM for R4 (PARAMETERIZED on access-need determination), MEDIUM for R6 (access management program scope)

---

## Scope summary

CIP-004 governs the human side of CIP: awareness, training, personnel risk assessments (PRAs), access authorization, access revocation, and (v7) the access management program. Most requirements have crisp cadences (15 months, 7 calendar days, 24 hours) that map directly to time-window assertions. The PARAMETERIZED elements are typically around "need" — who needs access, what training scope is appropriate for a given role.

"""
CIP-004-7 R2 — Cyber Security Training
Confidence: HIGH | Human Review: NOT REQUIRED
"""
from datetime import datetime, timedelta

REQUIRED_CONTENT_AREAS = {
    "Cyber security policies",
    "Physical access controls",
    "Electronic access controls",
    "Visitor control program",
    "Handling of BCSI",
    "Identification of a Cyber Security Incident",
    "Recovery plans for BCS",
    "Response to Cyber Security Incidents",
    "Cyber security risks associated with vendor remote access",
}

TRAINING_WINDOW_DAYS = 457  # 15 months


def test_CIP004_R2_training_before_access(personnel_with_hm_access):
    for p in personnel_with_hm_access:
        assert p.last_training_completed_at is not None, (
            f"VIOLATION: {p.person_id} has H/M access but no training record."
        )
        assert p.last_training_completed_at <= p.access_granted_at, (
            f"VIOLATION: {p.person_id} was granted access "
            f"{p.access_granted_at} before training {p.last_training_completed_at}."
        )


def test_CIP004_R2_training_content_areas(training_records):
    for record in training_records:
        missing = REQUIRED_CONTENT_AREAS - set(record.content_areas_covered)
        assert not missing, (
            f"VIOLATION: Training {record.record_id} missing required "
            f"content areas: {missing}"
        )


def test_CIP004_R2_training_recurrence(personnel_with_hm_access):
    now = datetime.utcnow()
    for p in personnel_with_hm_access:
        age = now - p.last_training_completed_at
        assert age <= timedelta(days=TRAINING_WINDOW_DAYS), (
            f"VIOLATION: {p.person_id} training is {age.days} days old "
            f"(threshold {TRAINING_WINDOW_DAYS})."
        )
```

"""
CIP-004-7 R5 — Access Revocation
Confidence: HIGH | Human Review: NOT REQUIRED
"""
from datetime import timedelta


def test_CIP004_R5_termination_revocation_24h(termination_events):
    """Electronic + unescorted physical access revoked within 24h."""
    for event in termination_events:
        for revocation in event.revocations:
            if revocation.access_type in (
                "Interactive Remote Access",
                "Unescorted Physical Access",
            ):
                delta = revocation.revoked_at - event.action_at
                assert delta <= timedelta(hours=24), (
                    f"CRITICAL VIOLATION: {event.person_id} "
                    f"{revocation.access_type} revoked "
                    f"{delta.total_seconds() / 3600:.1f}h after termination "
                    f"(threshold 24h)."
                )


def test_CIP004_R5_all_required_types_revoked(termination_events):
    required = {"Interactive Remote Access", "Unescorted Physical Access"}
    for event in termination_events:
        revoked_types = {r.access_type for r in event.revocations}
        missing = required - revoked_types
        assert not missing, (
            f"CRITICAL VIOLATION: {event.person_id} terminated but "
            f"access types not revoked: {missing}"
        )
```

## Notes for the registry

- **R3 nuance:** Identity verification and criminal-history-check presence are deterministic, but the entity's *decision* on the PRA result is parameterized. We test the presence of the decision and the named evaluator; we do not test the substantive judgment.
- **R5 audit hot-spot:** Late revocations are the single most common CIP-004 violation. The 24-hour clock is unforgiving — make sure HRIS-to-IAM provisioning latency is measured separately and alerted on.
